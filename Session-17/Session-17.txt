Task_1
Ένα κλειδί API και ένα access token εξυπηρετούν τον ίδιο σκοπό: είναι μοναδικά αναγνωριστικά που χρησιμοποιούνται για τον έλεγχο ταυτότητας κλήσεων σε ένα API.
Αποτελείται από μια σειρά γραμμάτων και αριθμών που προσδιορίζουν την client application που υποβάλλει το αίτημα,
ένα κλειδί API ή ένα access token χρησιμοποιείται για τη χορήγηση ή την απόρριψη αιτημάτων με βάση τα δικαιώματα πρόσβασης του client 
και την παρακολούθηση του αριθμού των αιτημάτων.

Task_2
Το Content-Type header χρησιμοποιείται για να υποδείξει τον τύπο μέσου του resource.
To media type είναι ένα string που αποστέλλεται μαζί με το αρχείο που υποδεικνύει τη μορφή του αρχείου.
Για παράδειγμα, για ένα αρχείο εικόνας, ο τύπος του μέσου θα είναι image/png ή image/jpg.

Το αίτημα GET θα πρέπει να λαμβάνει μόνο δεδομένα (ο διακομιστής δεν πρέπει να αλλάξει την κατάστασή του).

Task_6
400 Bad Request Permalink
Το μήνυμα σφάλματος 400 Bad Request είναι ένα από τους πιο γενικούς κωδικούς κατάστασης HTTP.
Σημαίνει ότι δεν μορφοποιήσατε σωστά το αίτημα API σας. Μπορεί να λείπει ένα query, ένα πεδίο στο σώμα του αιτήματος ή ένα header field μπορεί να είναι λάθος.
Μπορεί επίσης να είναι ότι ορισμένα από τα δεδομένα του αιτήματός σας μπορεί να έχουν λανθασμένη σύνταξη.

Task_7
404 Not FoundPermalink
Αυτός είναι μακράν ο πιο κοινός status code HTTP που μπορείτε να λάβετε.
Υποδεικνύει ότι η διεύθυνση URL που χρησιμοποιήσατε στο αίτημά σας δεν υπάρχει στον διακομιστή API ή στον διακομιστή προέλευσης.
Αν και αυτό είναι ένα σφάλμα 4XX, το οποίο συνήθως σημαίνει ότι κάτι στην πλευρά του client δεν πάει καλά, αυτό επίσης μπορεί να υποδηλώνει πρόβλημα στο server.
Μερικές φορές τα URL paths του API αλλάζουν μετά από μια ενημέρωση έκδοσης, αλλά μερικές φορές αλλάζουν επειδή κάτι στον διακομιστή πήγε στραβά.
